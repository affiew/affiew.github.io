哈希表 Hash Table

  • 用来快速判断一个元素是否出现集合里 O(1)

  • 哈希函数 Hash function：映射到哈希表上的索引

  • 哈希碰撞

    • 拉链法：选择适当的哈希表的大小，这样既不会因为数组空值而浪费大量内存，也不会因为链表太长而在查找上浪费太多时间。
    • 线性探测法：一定要保证tableSize大于dataSize。 我们需要依靠哈希表中的空位来解决碰撞问题。

  • 常见的三种哈希结构

    • array
    • set
    • map

  • 当我们遇到了要快速判断一个元素是否出现在集合里的时候，就要考虑哈希法

  • 牺牲了空间换取了时间

Leetcode 160. Intersection of Two Linked Lists

https://leetcode.com/problems/intersection-of-two-linked-lists/

  1. 暴力解法 没过
  2. 求两个链表长度差值 移动指针到末尾对齐
  • 时间复杂度：O(n + m)
  • 空间复杂度：O(1)

  1. 另一种解法

    https://leetcode.cn/problems/intersection-of-two-linked-lists/solution/intersection-of-two-linked-lists-shuang-zhi-zhen-l/

  • headA链表节点数量为a，headB链表节点数量为b，公共长度c

  • headA到相交节点：a-c，headB到相交节点：b-c

  • A , B分别指向两链表头节点，headA，headB：

    • A先遍历完链表headA ，再开始遍历链表headB ，当走到交点时，共走步数为：a+(b-c)
    • B先遍历完链表headB ，再开始遍历链表headA ，当走到交点时，共走步数为：b+(a-c)
    • A，B重合时：a+(b-c)=b+(a-c)
      • 若两链表 有 公共节点 (c>0) ：指针A，B同时指向相交节点
      • 若两链表 无 公共节点 (c=0) ：指针A，B同时指向None

Leetcode 142. Linked List Cycle II

https://leetcode.com/problems/linked-list-cycle-ii/

没想出来 只知道快慢指针找环不知道怎么找环的入口

随想录：

  • fast指针一定先进入环中，如果有环fast指针和slow指针一定是在环中相遇
  • 假设从头结点到环形入口节点为x，环形入口节点到相遇节点为y, 从相遇节点到环形入口节点为z

  • 相遇时： slow指针走过 x + y，fast指针走过 x + y + n (y + z)，n为fast在环内走了n圈， y+z为 一圈内节点的个数。

  • fast指针走过的节点数 = slow指针走过的节点数 * 2：(x + y) * 2 = x + y + n (y + z)

  • x = n (y + z) - y = (n - 1) (y + z) + z

  • n == 1 时 x = z

  • 头结点出发一个指针，相遇节点也出发一个指针，这两个指针每次只走一个节点，当这两个指针相遇的时就是环形入口节点。

https://leetcode.com/problems/binary-search/

  • [left, right] 要用 <=
  • right = middle - 1
  1. 左闭右开
